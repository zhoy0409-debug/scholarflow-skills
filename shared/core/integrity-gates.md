# 学术诚信门禁 —— 在动手分析之前跑，不是靠模型碰巧发现

> 来源：「Prostate cancer manuscript rejection」线程。
> 当时是靠模型**碰巧**注意到文件名里有 `Mock`、且 320×191 零缺失，才拦下来的。
> 「碰巧」不是门禁。写成清单，每次自动跑。

## 数据来源红旗（拿到任何数据文件先扫一遍）

扫到任何一条，**先停下来问用户，不要开始分析**：

- [ ] 文件名含 `mock` / `sim` / `simulated` / `fake` / `test` / `dummy` / `demo` / `example`
- [ ] **零缺失**。真实的问卷/病历采集几乎不可能 0 缺失。N×M 全满是生成数据的指纹。
- [ ] 样本量恰好等于某个整数或恰好命中 power 计算的下限（320、100、n=计算值+0）
- [ ] 连续变量分布过于对称、无异常值、小数位数整齐
- [ ] 组间基线**过度平衡**（真实随机化会有噪声）
- [ ] 有伦理批件号、基金号，但数据文件却是生成的 → 这是最危险的组合

问法要中性、给对方台阶：

> 「这份数据是真实采集的，还是先用模拟数据把分析流程跑通？
>  如果是后者，我可以帮你把代码和流程搭好，等真实数据到位直接套用——这完全没问题。
>  但如果要以真实研究去投稿，我需要先确认数据性质。」

如果用户确认是真实数据：**照做，但提醒他改掉文件名、留好原始录入记录**。
「Mock」这个词和「零缺失」正是期刊诚信审查最先盯的两点——真数据也别留这种误解口。

## 结果自洽红旗（拿到别人给的实验报告先扫一遍）

> 来源：「Q-PCR results evaluation」线程。外包公司报了「约 2 倍上调」，
> 但原始 Ct 里目的基因根本没前移，GAPDH 内参在过表达组整体晚了 1.5 个 Ct。
> **那 2 倍是内参漂移校正出来的，不是真的过表达。**

拿到 qPCR / ΔΔCt 结果时必查：

- [ ] **内参 Ct 是否跨组一致**。内参漂移 > 0.5 Ct → 上样量不等，任何 ΔΔCt 结论都不成立
- [ ] **目的基因的原始 Ct 有没有朝预期方向移动**。过表达组的目的基因 Ct 必须**前移**。
      若原始 Ct 不动甚至升高、而报告说上调 → 结论完全来自归一化，无效
- [ ] 引物是否真的落在构建的插入序列上（拿到 .gb / 质粒图自己比对）
- [ ] 靶标搞错没有（`Cx3cl1`(配体) vs `Cx3cr1`(受体) 这类一字之差）
- [ ] 有没有熔解曲线、扩增曲线、每孔原始 Ct。**只给汇总倍数不给原始 Ct 的报告，不接受**

同类的，别的 assay：
- WB：有没有 loading control，曝光是否过饱和
- 测序：Q30、覆盖度、比对率
- 流式：有没有 FMO / isotype 对照
