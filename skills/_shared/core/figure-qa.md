# 出图 QA 环 —— 「代码跑通」不等于「图做好了」

> 来源：用户上传的 Codex 版 `polish-sci-figures`。
> 那个 skill 整体不该采用（description 是典型的关键词堆砌，会和 nature-figure /
> paper2ppt 直接对撞；192 行单体、无渐进披露、无 manifest）。
> 但它有四样东西是被真实踩坑打磨出来的，值得整段吸收。这是其中最重要的一条。

## 核心那一句

> **Do not equate successful code execution with a finished figure.**
> 代码没报错 ≠ 图是对的。

matplotlib 不会因为 label 被裁掉而报错，不会因为 panel 标号错位而报错，
不会因为图例压在数据上而报错。**必须真的把 PNG 打开看一眼。**

## 强制 QA 环（出图后必跑，不许跳）

1. 从**权威源**出图 —— 先确认哪个文件是真正被嵌进稿子/PPT 的那个。
   别精修一个已经废弃的中间产物。
2. 导出高分 PNG 预览 + SVG/PDF。
3. **打开每一张最终 PNG 用眼睛看**。多图时再拼一张 contact sheet 看整体一致性。
4. **按最终插入尺寸看，不要只看放大版**。A4 页面 100% 缩放下每个必要标签都得看得清；
   投影幻灯上从观众席得看得清。
5. 查科学映射：分组、方向、标签、单位、样本量、显著性、图例、panel 顺序。
6. 查几何：对齐、等间距、标号网格、页边距、留白、panel 平衡、长宽比。
7. 查缺陷：裁切、重叠、重复文字、字太小、缺轴、悬空轴、断线、残留旧标签、
   配色漂移、乱码字形。
8. **把图真的嵌进 DOCX/PPT，渲染出真实页面/幻灯，再看一遍。**
   单看一张图没问题，插进去之后翻车是常态。
9. 每发现一个问题就修，然后**重跑相关检查**。

## Release blockers（任何一条命中，不许交付）

```
BLOCK  panel 标号偏离共享网格（同行必须同一个 y，同列必须同一个 x）
BLOCK  任何必要文字被裁切、重叠、或小到读不清
BLOCK  无法解释的大片空白，或某个 panel 明显脱离版面
BLOCK  缺轴 / 悬空轴 / 标注挂错了点
BLOCK  字体、标号尺寸、语义配色不一致
BLOCK  纯位图产物被描述成「可编辑」
BLOCK  最终文档没有被渲染出来看过
```

## 可编辑性诚信（这条要写死）

- 优先级：**绘图脚本+源数据 > 原生矢量(AI/SVG/可编辑PDF/PPT形状) > 高分位图**
- 有原生源就绝不去 patch 一张压平的 PNG
- 只有位图时，**必须明说「原始像素不可编辑」**，新加的文字/标注层可编辑，
  产物报告成「部分可编辑」
- **绝不能声称「把 PNG 套进 SVG 里」就让底层图元可编辑了。** 这是谎话。

## 图形形式的选择（反「满屏柱状图」）

一份全是可互换柱状图的稿子/PPT，说明作者没在想「这张图要说什么」。
按**结论类型**选形式：

| 要讲的 | 用 | 不要用 |
|---|---|---|
| 分布 | 散点 + box/violin/raincloud | 光柱子 |
| 配对样本 | 配对点 / 斜线图 | 两根并排的柱子 |
| 上调/下调方向 | 箭头 / 双向柱 / lollipop | 单向柱 |
| 效应量排序 | lollipop / 横向柱 | 竖柱 |
| 多特征 × 多条件 | 热图 / 点阵图 | N 个子柱图 |
| 富集分析 | 点图 / lollipop | 柱图 |
| 时间序列 | 折线 + 不确定性带（标签不许压在线上） | 柱 |
| 效应估计 + 不确定性 | 森林图 | 柱 + 误差棒 |
| 网络拓扑 | 网络图（**只在拓扑本身是结论时**） | — |

原则：**可比的 panel 保持可比。** 别为了花哨而换形式，那会破坏跨 panel 阅读。

## 科学意义保护

- 出图前先核：比较对象、样本量、**生物学重复 vs 技术重复**、组别方向、统计检验、单位、
  标签到数据的映射。
- **绝不编造**数值、显著性、标签、趋势、误差棒、缺失的观测。
- 旧图在科学上就是错的 → **从源数据重画**，不要因为「原文就长这样」而保留错误。
- 显微/WB/结构图的图像内容不许改（除了明示的、不误导的排版和标注）。标尺必须留。
- 每一个颜色、线、箭头、标注、高亮点**都必须有定义好的含义**。装饰性的删掉。

## 一处不采纳

Codex 那份把 biomedical manuscript figure 的默认字体定为 **Times New Roman**。
这个默认是错的 —— 绝大多数期刊的**图内文字要求无衬线**（Arial/Helvetica），
正文才用衬线。默认应为 **Arial**，除非目标期刊/模板另有规定。
（你自己在「Figure formatting adjustments」那条线程里，最后也是统一成 Arial 的。）
