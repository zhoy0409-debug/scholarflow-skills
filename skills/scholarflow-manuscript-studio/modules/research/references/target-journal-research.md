# Target Journal Research

Use this before writing for any named journal or conference.

## Purpose

Prevent venue-blind writing. A manuscript cannot be "high quality" for a target venue until the skill has learned the venue's official rules and recent article behaviour.

## Search Requirements

For a named venue:

1. Open the official author guidelines.
2. Record article type, page/word limit, section sequence, abstract requirements, data/software availability, AI policy, and template requirements.
3. Find 5-10 recent papers in the same venue and adjacent topic.
4. Pick 3-6 papers for style learning.
5. State why each selected paper is a useful model.

Use current official web sources when available. Local prior notes may support the analysis but do not replace current venue research.

## Venue-Specific Deep Checklist

For the named venue, verify current official requirements rather than relying on memory. Usually check:

- Article-type page limit and approximate word count.
- Structured abstract headings.
- Required or expected sections.
- Data availability statement.
- Software/source-code availability for computational methods.
- Cover-letter suitability argument.
- LaTeX template requirements.
- AI-use disclosure requirements.
- Whether the manuscript fits the selected article type.

Then inspect recent venue papers in the user's topic area and learn:

- how the opening states the scientific question,
- how Results summarizes advance and impact,
- how Introduction moves from field context to specific gap,
- how Methods justify design choices,
- how Results sections order benchmark, comparison, generalization, interpretation, and domain analysis,
- how Discussion resolves the motivation and acknowledges limitations.

Record these observations as venue logic patterns. Later `paragraph_function_templates.md`, `result_narrative_templates.md`, and `section_blueprints.md` must cite these patterns when the user targets the named venue.
