## Pre-Phase 0: Extract the Motivation Thread

Before rewriting any text, extract the central motivation from the original draft.

**Step 1 — Identify take-home messages.** Read the Discussion/Conclusion (if it exists) or the Abstract. List the 3-5 core claims. These are your "punchlines" (Montagnes et al.'s "writing backwards" method).

**Step 2 — Map the motivation across sections.** For each section:
| Section | What role does it play in the motivation? | Is this role actually fulfilled? |
|---------|------------------------------------------|----------------------------------|
| Abstract | Distills full motivation arc (problem → approach → solution) | |
| Introduction | Establishes why problem matters, identifies gap, states research question | |
| Methods | Justifies how the approach CAN solve the problem | |
| Results | Presents evidence that leads to the solution | |
| Discussion | Resolves motivation: "here is the answer, and here is why it matters" | |

**Step 3 — Identify motivation gaps.** Common issues:
- Introduction drifts into general background without narrowing to a specific gap
- Methods read like a recipe without explaining WHY each step matters
- Results report data without connecting findings back to motivation
- Discussion restates results instead of resolving the problem from Introduction

**Step 4 — Write the one-sentence "red thread".** Produce: *"This paper [action] [specific problem] because [motivation: why it matters], and finds that [core take-home]."* This sentence is the yardstick for every paragraph.

## Phase 1: Move-Guided Section Rewrite

**Before starting**: Review Section 7 (Move Structure Consensus) from Cross-Paper Synthesis. Keep the "red thread" visible.

For each section, execute:

**Step A — Map the original section's moves.** For each paragraph, identify which rhetorical move it performs (using Template H's move taxonomy). Record: which obligatory moves are present, which are MISSING, which paragraphs combine multiple moves.

**Step B — Map the reference papers' moves.** Using move maps from Template H, confirm the obligatory move sequence and how each paper executes each move (paragraph count, sentence count, opening/closing patterns).

**Step C — Build the move-level rewrite plan.** For each obligatory move:
- **Our paragraph #**: which paragraph executes this move
- **Model paper**: which of the 3 papers executes this move best
- **Content source**: what content from original draft fills this move (NO new data)
- **Key language**: verbatim transition/openers from model paper to reuse structurally

**Step D — Rewrite move by move.** Write each move as a unit:
- Open the move the same way the model paper does (reuse structural pattern, not words)
- Match data density (if model paper's equivalent move has no numbers, neither should ours)
- Maintain claim strength consistent with model paper
- After each move, verify: does this paragraph fulfill the move's communicative purpose?

**Step E — Section-level check.** After rewriting all moves:
1. **Move coverage**: All obligatory moves present?
2. **Move order**: Sequence matches consensus?
3. **Motivation check**: Does this section advance the red thread?
4. **Stylistic fit**: If inserted among 3 reference papers, does it blend in?
5. **Numerical audit**: Trace every number to its source (Principle 6). Remove fabrications.

## Phase 2: Structural Coherence Pass

After all sections rewritten, read the full draft. Fix:
- Cross-section flow (does Introduction's last paragraph lead into Methods?)
- Transition consistency (same patterns throughout?)
- Terminology consistency (same term for same concept everywhere?)

## Phase 3: Numerical Audit (Mandatory)

Extract every numerical value from the revised draft. For each:
- **Source**: original draft / deep-read paper N / neither
- If "neither" → **DELETE** and replace with qualitative language or `[NEED DATA]` placeholder
- If from original draft → verify value was not altered

Save audit table in `revision_1_literature.md`.

## Phase 4: Cross-Section Motivation Audit

After all sections rewritten and numerically audited, verify the motivation thread.

**Audit 1 — Problem–Solution arc.** Read these three sentences in sequence:
- Last sentence of Introduction (research question / purpose statement)
- First sentence of Discussion (what did we find?)
- Last sentence of Discussion (why does it matter?)

Do they form a coherent arc? Introduction states the problem; Discussion resolves it.

**Audit 2 — Introduction↔Discussion pairing** (Lingard, 2017). List every major claim from Introduction. Find where addressed in Discussion:
| Introduction claim/question | Resolved in Discussion (paragraph #) | Status |
|---------------------------|--------------------------------------|--------|
| [claim 1] | P# | resolved / partially / NOT ADDRESSED |

Any "NOT ADDRESSED" → broken thread. Add to Discussion or remove from Introduction.

**Audit 3 — Parallel structure.** Verify same theme sequence across sections:
- List sub-themes in Introduction (in order)
- List sub-themes in Results (in order)
- List sub-themes in Discussion (in order)
- Do they follow the same sequence?

**Audit 4 — Motivation lineage.** Collect the ONE sentence per section that most directly connects to the central motivation:
| Section | Motivation-anchoring sentence |
|---------|------------------------------|
| Abstract | |
| Introduction P1 (opening) | |
| Introduction last P (research question) | |
| Methods first P | |
| Results first P | |
| Discussion first P | |
| Discussion last P (closing) | |

Read these 7 sentences in sequence. They should tell the complete story. If any feels disconnected, rewrite it.
